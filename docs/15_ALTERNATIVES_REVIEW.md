# 15. Обзор готовых self-hosted решений

## 1. Вывод

Подобные решения существуют, но обычно они закрывают только часть задачи:

- Nextcloud — универсальное облако;
- Immich — фото/видео как Google Photos;
- Seafile — быстрая файловая синхронизация;
- Syncthing — P2P-синхронизация;
- OpenMediaVault — NAS-панель;
- PhotoPrism — фотоархив/каталог;
- TrueNAS — мощный NAS, но не для Jetson Nano;
- Open WebUI/Ollama — локальный LLM-интерфейс, но не для Stage 1 на Jetson Nano.

Отличие данного проекта — не в уникальности каждого компонента, а в интеграционном шаблоне: **семейное облако + фотоархив + Android restore roadmap + DeepSeek LLM Gateway + Codex-ready инженерная документация**.

## 2. Сравнительная таблица

| Решение | Категория | Сильные стороны | Ограничения | Роль в нашем проекте |
|---|---|---|---|---|
| Nextcloud | Облако/PIM | файлы, WebDAV, календарь, контакты, пользователи, приложения | требует настройки и ресурсов | основной компонент |
| Immich | Фото/видео | мобильный backup, web-галерея, UX как Google Photos | требует RAM, ML тяжёлый | основной компонент, ML off |
| Seafile | Файлы | быстрая синхронизация, эффективный sync | меньше PIM/экосистемы | альтернатива Nextcloud для файлов |
| Syncthing | P2P sync | простота, нет центрального сервера | нет полноценного облака и пользователей | доп. синхронизация папок |
| OpenMediaVault | NAS OS | web-панель, SMB/NFS/FTP/SSH, ARM | может конфликтовать с кастомным stack на Jetson | альтернатива ручному NAS-слою |
| PhotoPrism | Фотоархив | каталогизация старых библиотек, метаданные | тяжеловат, нет focus на mobile backup | альтернатива/Stage 3 |
| Piwigo/Lychee | Web-галерея | лёгкие галереи | не полноценный mobile backup | не основной выбор |
| TrueNAS | NAS | ZFS, enterprise-функции | нецелесообразен для Jetson Nano | не использовать на данном железе |
| Open WebUI | LLM UI | универсальный интерфейс LLM | локальная LLM не используется | Stage 3, не Stage 1 |
| Home Assistant | Home automation | умный дом | не решает облако/backup | вне ядра проекта |

## 3. Почему выбраны Nextcloud и Immich

Nextcloud закрывает PIM и файловое облако. Immich лучше подходит для фото/видео с телефонов. В паре они дают функциональность, близкую к Google Drive + Google Photos, но на собственном сервере.

## 4. Почему не Seafile как основной

Seafile хорош для файловой синхронизации, но хуже подходит как семейная платформа с календарями, контактами и расширяемыми приложениями.

## 5. Почему не только Syncthing

Syncthing отлично синхронизирует папки, но не даёт web-облако, пользователей, семейные shared folders, календарь и контакты.

## 6. Почему не OpenMediaVault как ядро

OpenMediaVault удобен как NAS-панель и официально ориентирован на NAS-сценарии с ARM-поддержкой. Но в данном проекте требуется гибкая сервисная архитектура с Nextcloud, Immich, будущим Android API и LLM Gateway. Поэтому OMV оставлен как альтернативный путь.

## 7. Возможность публичного развития

Проект может занять нишу:

```text
Home cloud blueprint for old SBC/ARM boards, focused on family Android backup and privacy-controlled AI administration.
```

Чтобы проект «взлетел», нужно добавить:

1. реально проверенные install scripts;
2. поддержку не только Jetson Nano, но и Raspberry Pi 4/5, mini-PC, старых ноутбуков;
3. wizard для `.env`;
4. CI-проверки YAML/scripts;
5. подробные скриншоты;
6. инструкции для Android/Xiaomi/HyperOS;
7. демонстрационный режим без реальных секретов;
8. понятную лицензию и CONTRIBUTING.
