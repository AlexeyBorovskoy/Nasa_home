# ADR-0002: Дизайн хранилища — USB HDD + ext4 + /mnt/storage

## Статус

Принято.

## Контекст

Jetson Nano не имеет встроенного HDD/SSD. Проект требует нескольких сотен ГБ для фото/видео (Immich), файлов (Nextcloud) и БД.

## Рассмотренные варианты

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| microSD только | Уже есть | Ресурс 10K циклов, медленно для БД, мало места |
| **USB HDD (внешнее питание)** | Дёшево, большой объём | Требует питания |
| USB SSD | Быстро, надёжно | Дороже |
| NFS | Масштабируется | Зависимость от второго устройства |
| eMMC | Встроен | Только 16 GB |

## Решение

USB HDD с отдельным блоком питания, **ext4**, смонтирован как `/mnt/storage`.

Структура:

```
/mnt/storage/
├── nextcloud/data/          # файлы пользователей
├── immich/library/          # фото и видео
├── db/
│   ├── nextcloud-postgres/
│   └── immich-postgres/
├── backups/
│   ├── database-dumps/      # pg_dump, gzip, ротация 7 шт.
│   └── restic-repo/
└── samba/
```

Монтирование через `/etc/fstab` по UUID с `defaults,nofail`.

## Временная конфигурация Stage 1

USB-флешка вместо HDD — для проверки Docker Compose и путей без риска для реальных данных.

## Последствия

- `fstab` должен иметь `nofail` — Jetson загружается без диска.
- `scripts/diagnostics/storage_health.sh` проверяет монтирование перед операциями.
- Все пути задаются через `config/.env` (STORAGE_ROOT, NEXTCLOUD_DATA и т.д.).
