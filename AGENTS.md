# AGENTS.md

## 1. Общие правила для Codex/агентов

Агент работает как инженер-разработчик и системный администратор. Основная цель — безопасно и пошагово разворачивать домашнюю облачную платформу.

## 2. Жёсткие ограничения

1. Не записывать реальные пароли, API-ключи, токены и персональные данные в репозиторий.
2. Не выполнять destructive-команды без явного подтверждения пользователя:
   - `rm -rf`;
   - форматирование дисков;
   - изменение таблицы разделов;
   - очистка Docker volumes;
   - удаление БД;
   - изменение firewall/VPN на боевом роутере.
3. Не открывать Nextcloud, Immich, LLM Gateway или SSH напрямую в интернет без отдельного документа риска.

## 2а. Сетевые ограничения (критично)

- **Никогда не трогать Amnezia-сервер на EU VPS через SSH или `wg set`** — это вызывает рестарт контейнера `amnezia-awg` и роняет ~25 VPN-клиентов (телефоны семьи). Единственный безопасный способ — десктоп-приложение Amnezia.
- **Профиль `nasa-lan` на Jetson не удалять** — это рабочая статическая конфигурация eth0 (`192.168.0.50/24`). Предложения «вернуть DHCP» — ошибочны.
- **Для внешнего доступа использовать Tailscale** (ADR-0004, `docs/plans/TAILSCALE_ACCESS_PLAN.md`) — пробивает CGNAT, не трогает Amnezia.
- Подробнее: ADR-0003 (`docs/decisions/ADR-0003-networking-lan-only.md`).
4. Не отправлять личные фото, видео, контакты, календарь, документы или backup-манифесты во внешний LLM.
5. На первом этапе не разворачивать локальную LLM на Jetson Nano.

## 3. Рабочий процесс

Каждое изменение должно идти по схеме:

```text
1. Что меняется.
2. Почему меняется.
3. Какие файлы затрагиваются.
4. Команды.
5. Проверка результата.
6. Rollback.
```

## 4. Формат результата агента

Агент должен возвращать:

- краткое резюме;
- список изменённых файлов;
- команды запуска/проверки;
- риски;
- следующий шаг.

## 5. Правило малых шагов

Один шаг — один технический блок. После каждого шага должен быть контроль результата.
