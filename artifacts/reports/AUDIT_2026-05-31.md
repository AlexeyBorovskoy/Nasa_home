## 2. Что готово к запуску (Stage 1)

1. `cp config/.env.example config/.env && chmod 600 config/.env`
2. Заполнить все `CHANGE-ME`/`change_me`/`replace_me` (хотя бы DB-пароли и DEEPSEEK_API_KEY).
3. Подготовить HDD: `docs/04_STORAGE_DESIGN.md` (ext4, mount, fstab, структура каталогов).
4. `docker compose -f docker/compose/docker-compose.stage1.yml --env-file config/.env up -d`
5. Проверки: `scripts/diagnostics/{hardware_audit,docker_health,storage_health}.sh`.

## 5. Переменные в `config/.env.example`, не используемые в коде

Эти переменные оставлены как «зарезервировано на будущее» или для документации,
но прямо в compose/скриптах сейчас не читаются — можно либо реализовать, либо
удалить:

- `PUID`, `PGID` — типично для Linuxserver-образов; в наших compose не нужны.
- `JETSON_LAN_IP`, `NEXTCLOUD_HOST`, `IMMICH_HOST`, `LLM_GATEWAY_HOST` — только
  для reverse-proxy секции (Caddy/Traefik), которой пока нет.
- `IMMICH_DISABLE_MACHINE_LEARNING=true` — должен быть передан в compose как env
  для `immich-server`/`immich-machine-learning`, сейчас не передаётся.
- `POSTGRES_NEXTCLOUD_PASSWORD` — дубликат `NEXTCLOUD_DB_PASSWORD`, нигде не
  читается. Удалить или переименовать.
- `LLM_DAILY_TOKEN_LIMIT`, `LLM_MONTHLY_COST_LIMIT_USD`,
  `LLM_ALLOW_FILE_CONTENT_UPLOAD`, `LLM_ALLOW_IMAGE_ANALYSIS` — для policy
  enforcement в LLM Gateway; пока не имплементировано в `services/llm-gateway/app/main.py`.
- `OPENAI_BASE_URL`, `OPENAI_API_KEY`, `OPENAI_MODEL` — fallback-провайдер, в
  коде сейчас только DeepSeek-ветка.
- `BACKUP_RETENTION_*` — передаются через `restic forget` в скрипте напрямую
  цифрами, не через env.

**Рекомендация:** на Stage 2 либо реализовать чтение этих переменных, либо
вычистить из шаблона.
